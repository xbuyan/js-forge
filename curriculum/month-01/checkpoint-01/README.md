# Checkpoint 1: Variables & Types Assessment

## Self-Evaluation

Before proceeding, honestly assess yourself:

```
□ I can explain the difference between var/let/const with concrete examples
□ I can predict hoisting behavior in 5/5 test cases
□ I can identify and fix type coercion bugs
□ I can write a script that runs in both Node.js and browser
□ I can safely parse and validate user input
□ I understand the Temporal Dead Zone
□ Time taken: ___ hours | Confidence (1-10): ___
```

## Practical Challenge

Build a configuration validator that:
1. Reads environment variables and JSON files
2. Validates types against a schema
3. Prevents prototype pollution
4. Returns detailed error messages

## Peer Review Prompts

Review a peer's solution and check:
- [ ] Proper use of const vs let
- [ ] No var usage in modern code
- [ ] Hoisting awareness demonstrated
- [ ] Type coercion handled safely
- [ ] Error messages are helpful

## Time Estimate
- Challenge: 2-3 hours
- Peer review: 30 minutes
